Deploy to Render (Recommended)

Render provides: Free tier, automatic HTTPS, environment variables, continuous deployment

Quick Deploy Steps:

  1. Click "New +" ‚Üí "Web Service" at render.com
  2. Connect your GitHub repository
  3. Render will auto-detect the render.yaml configuration
  4. Add environment variables in Render dashboard:
    GOOGLE_CLIENT_ID=your_client_id
    GOOGLE_CLIENT_SECRET=your_client_secret
    AUTHORIZED_EMAILS=hannes.mitterer@gmail.com,alfred.mitterer@gmail.com,...
  5. Update Google OAuth redirect URI to: https://your-app.onrender.com/auth/google/callback
  6. Click "Create Web Service"

Your app will be live at: https://seedbringer-interface.onrender.com

Deploy to Netlify (Alternative)

Note: Netlify is best for static sites. For full OAuth functionality, use Render.

Static Deployment (Information Only):

  1. Go to netlify.com
  2. Click "Add new site" ‚Üí "Import an existing project"
  3. Connect GitHub and select your repository
  4. Netlify will use the netlify.toml configuration
  5. Click "Deploy site"

Note: This deploys only static files (GitHub Pages style). Backend features require serverless functions or use Render instead.

Full Application Access

To access the full OAuth-authenticated communication interface:

  1. Deploy to Render using instructions above
  2. Configure Google OAuth credentials
  3. Navigate to your deployed URL
  4. Sign in with authorized Gmail account
